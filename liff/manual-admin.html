เพิ่มพนักงานใหม่ (Onboard)

มี 2 วิธี:

วิธี A: พนักงาน register เอง (ขั้นตอนเริ่มต้น)

  1. พนักงานเปิด LIFF → กด "ลงทะเบียน" → กรอกฟอร์ม
  2. คุณ (HR) ได้ Flex card แจ้งใน LINE
  3. กดปุ่ม "อนุมัติ" ใน flex หรือเปิด admin → แท็บ "พนักงาน" → กด ⚙️ → อนุมัติ
  4. ระบบสร้าง LeaveQuota row ตาม default ให้อัตโนมัติ

วิธี B: HR เชิญ (สำหรับเคสที่อยาก pre-onboard)

  1. แท็บ "พนักงาน" → "+ เพิ่มพนักงานใหม่"
  2. กรอกข้อมูล → ระบบสร้าง pairing code 6 หลัก + ข้อความเชิญ
  3. คัดลอกข้อความ ส่งให้พนักงานผ่าน LINE
  4. พนักงานกรอกรหัสในหน้า register → HR กด approve อีกครั้ง
ผูกหัวหน้างาน ↔ ลูกน้อง
  1. แท็บ "หัวหน้างาน" → "+ ผูกหัวหน้างาน"
  2. เลือกลูกน้อง (พนักงานที่ยังไม่มีหัวหน้า)
  3. เลือกหัวหน้างาน (ระบบจะ set is_supervisor=TRUE ให้อัตโนมัติ)
  4. กด "ผูก"

เปลี่ยนหัวหน้าได้ตลอด — ระบบเก็บประวัติทั้งหมด

ตั้งโควตาวันลาต่อพนักงาน

Default ทั้งระบบ ตั้งใน แท็บ "Settings":

  • default_sick_total — ลาป่วย (default 30 วัน/ปี)
  • default_personal_total — ลากิจ (default 6)
  • default_vacation_total — ลาพักร้อน (default 10)

Per user override — แท็บ "โควตา":

  1. เลือกพนักงาน
  2. กรอกโควตาที่ต้องการ (ทั้ง 3 ประเภท)
  3. กดบันทึก
HR เปลี่ยนได้ → ส่งให้เจ้าของอนุมัติก่อน apply
เจ้าของเปลี่ยน → apply ทันที
แก้กฎเงื่อนไขการลา
  1. แท็บ "กฎ" → "+ เพิ่มกฎใหม่" หรือกดแก้ rule เก่า
  2. ตั้ง:
    • แจ้งล่วงหน้า (วัน) — sick = 0 (ฉุกเฉิน), ลากิจ/พักร้อน = 3-7
    • ลาติดกันได้สูงสุด (0 = ไม่จำกัด)
    • ต้องแนบเอกสารถ้าลาเกิน N วัน
    • หมายเหตุ (แสดงให้พนักงานดูในหน้าลา)
  3. กดบันทึก
อนุมัติใบลา
  1. เมื่อมีใบลาเข้าระบบ → คุณได้ Flex card ใน LINE
  2. กดปุ่ม "อนุมัติ" หรือ "ไม่อนุมัติ" ใน flex โดยตรง
  3. หรือกดเมนู "อนุมัติ" ที่ rich menu → list ทุกใบที่รออยู่

HR กดอนุมัติชั้น 2 — ระบบส่งต่อไปเจ้าของชั้น 3
เจ้าของกดอนุมัติชั้น 3 — final approved + หักโควตาทันที

อนุมัติการเปลี่ยนแปลงของ HR (Owner only)

เมื่อ HR เสนอเปลี่ยน Settings / โควตา / กฎ → คุณต้องอนุมัติก่อน apply

  1. แท็บ "รออนุมัติ" (เห็นเฉพาะ OWNER)
  2. ดูรายละเอียดที่ HR เสนอ
  3. กด อนุมัติ / ปฏิเสธ
ดู Report
  1. แท็บ "รายงาน"
  2. filter ตามสถานะ (รออนุมัติ / อนุมัติแล้ว / ปฏิเสธ)
  3. กด "ค้นหา"
พนักงานออก (Offboard)
  1. แท็บ "พนักงาน" → กด ⚙️ ของพนักงานคนนั้น
  2. กด "ระงับการใช้งาน"
  3. ระบบ:
    • เปลี่ยน status = inactive
    • ถ้าเป็นหัวหน้างาน → ยกเลิก pair ของลูกน้องทั้งหมด (ต้อง pair ใหม่)
    • เก็บประวัติ + Audit_Log ครบ
